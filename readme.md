Este repositório é o meu espaço de estudo e prática em C# focado em:

- Problemas do LeetCode e exercícios de programação competitiva
- Estruturas de dados (listas, árvores, matrizes, etc.)
- Algoritmos clássicos (ordenação, busca, divisão e conquista, programação dinâmica)

Meu objetivo é reunir soluções bem comentadas, implementações eficientes e pequenos projetos/experimentos para facilitar revisão e aprendizagem.

---

## Conteúdo e estrutura

Principais pastas e arquivos:

- `LEETCODE/` — soluções para problemas do LeetCode e desafios (ex.: `buyAndSellStock.cs`, `twosums.cs`).
- `SORTING/` — implementações e estudos sobre algoritmos de ordenação (ex.: `quickSort.cs`, `sortColors.cs`).
- `LISTAS LP I E II/` — exercícios sobre listas e estruturas básicas.
- `PROGRAMAÇÃO COMPETITIVA/` — resoluções de problemas usados em competições e treinos (organizados por ano).
- `TesteApp/` — pequeno projeto .NET para testar/executar exemplos.

## Como usar

Pré-requisitos:

- .NET SDK (versão compatível com os projetos; a solução inclui um exemplo em .NET 9.0)

Exemplo rápido para compilar o projeto de exemplo usando PowerShell:

```powershell
cd "c:\Users\Tauan Neres\Documents\Projetos\Cursos\C#\TesteApp"
dotnet build
dotnet run
```

Para executar ou testar arquivos individuais (ex.: arquivos de solução de problemas), crie um projeto de console ou adicione o arquivo ao `TesteApp` e chame o método `Main` apropriado.

## Padrão de contribuição (simples)

1. Fork do repositório.
2. Crie uma branch com prefixo `feat/`, `fix/` ou `chore/` seguido de um resumo curto (ex.: `feat/two-sum-cleanup`).
3. Adicione o seu arquivo ou melhoria com comentários e testes simples quando possível.
4. Abra um Pull Request descrevendo a mudança e o problema/solução.

Recomendações de estilo:

- Nome dos arquivos: `camelCase` ou `PascalCase` seguindo o padrão C# (`twoSums.cs`, `QuickSort.cs`).
- Comente a intenção e a complexidade temporal/espacial (quando aplicável).
